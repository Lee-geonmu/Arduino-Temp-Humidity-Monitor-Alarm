# 아두이노 온습도 모니터링 및 스마트 경보 시스템

DHT11 센서로 온습도를 측정하여 LCD에 실시간 표시하고, 설정한 임계값 초과 시 LED와 부저로 위험을 알리는 아두이노 기반의 임베디드 프로젝트입니다. 센서 데이터의 신뢰성을 높이기 위해 소프트웨어 필터링과 디바운싱 로직을 적용했습니다.

## 🛠️ 주요 기능
- **실시간 모니터링:** 16x2 LCD를 통한 현재 온도 및 습도 출력
- **노이즈 제거:** 이동평균 필터(5개 샘플)를 적용하여 센서 튀는 값 방지
- **초기화 작업:** 부팅 시 센서 안정화를 위한 캘리브레이션 단계 수행
- **오작동 방지 (디바운싱):** 일시적인 수치 상승으로 인한 오작동을 막기 위해, **3회 연속 임계값 초과** 시에만 알람 발동 (임계값: 온도 30°C / 습도 70%)
- **예외 처리:** 센서 읽기 실패(Timeout, Checksum 에러 등) 시 LCD에 에러 메시지 표시

## 사용 부품 및 핀 맵

### 사용 부품
- Arduino Uno (1개)
- DHT11 온습도 센서 (1개)
- 16x2 I2C LCD (또는 일반 LCD)
- LED (1개), 부저 (1개), 저항 및 브레드보드

### 핀 연결 정보
| 부품 | 아두이노 핀 | 비고 |

| **DHT11 센서** | 7 | 데이터 핀 |

| **16x2 LCD** | 12, 11, 5, 4, 3, 2 | RS, E, D4, D5, D6, D7 |

| **경고 LED** | 13 | 디지털 출력 |

| **경고 부저** | 8 | 디지털 출력 |

## 트러블슈팅 (Troubleshooting) & 의사결정

### 문제 상황: 일시적 노이즈로 인한 경보 시스템의 오작동
* **현상:** 초기 프로토타입에서는 온습도 수치가 임계값(온도 30°C, 습도 70%)을 단 1회만 초과해도 즉시 LED와 부저가 작동하도록 구현했습니다.
* **원인:** 실제 환경에서 테스트한 결과, 센서 주변의 일시적인 기류 변화나 저가형 DHT11 센서 자체의 하드웨어적 노이즈로 인해 데이터가 순간적으로 튀는 현상이 발생했습니다. 이로 인해 위험 상황이 아님에도 경보가 울리는 **오작동** 문제가 발생하여 시스템의 신뢰성이 떨어졌습니다.

### 해결 방안: 소프트웨어 디바운싱 로직 도입
* **개선:** 데이터가 한 번 기준치를 넘었다고 바로 알람을 울리는 대신, 연속으로 3회 측정했을 때 모두 임계값을 초과하는 경우에만 실제 위험 상황으로 판단하도록 로직을 변경했습니다.
* **결과:** 일시적인 튀는 값이나 센서 노이즈를 성공적으로 걸러내어, 시스템이 정말로 지속적인 고온·고습 상태일 때만 정확하게 경보를 울리도록 **신뢰성을 대폭 향상**시켰습니다.
